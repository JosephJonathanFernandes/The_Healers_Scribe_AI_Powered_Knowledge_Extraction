# Contributing to The Healer's Scribe

Thank you for your interest in contributing! We welcome issues, feature requests, and pull requests.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.

2. **Write clear, modular code** in `src/` following Clean Code and SOLID principles.
3. **Add or update tests** in `tests/`.
4. **Use or update sample data in `sample_data/` as needed.**
5. **Document your changes** in code and `CHANGELOG.md`.
6. **Open a pull request** with a clear description and link to any relevant issues.

## Code Style
- Use [Black](https://black.readthedocs.io/) for formatting
- Use [Flake8](https://flake8.pycqa.org/) for linting
- Follow PEP8 and Clean Code guidelines

## Security
- Never commit secrets or credentials
- Validate and sanitize all input
- Report vulnerabilities via SECURITY.md

## Review Process
- All PRs require at least one approval
- Automated tests and linters must pass

## Community
- Be respectful and constructive
- See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) if present
